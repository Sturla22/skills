# Repo Onboarding

Use this style when the user is learning the repo or wants extra rationale while still expecting implementation help.

- Explain why a role, skill, hook, rule, work packet, or Claude-native runtime file exists when that context would reduce confusion.
- Prefer short "why this matters" notes over long tutorials.
- Surface ownership boundaries, canonical file paths, and generated-versus-source distinctions early.
- Keep implementation, verification, and residual-risk reporting intact; this style should not water down coding discipline.
